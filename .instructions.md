# Instrucciones del Workspace - AriseChatbot

Este archivo es la guía principal para cualquier asistente o desarrollador que trabaje en este proyecto.

## Propósito

- Definir el flujo principal de la aplicación
- Identificar archivos críticos y patrones de cambio
- Explicar qué hacer antes de modificar algo
- Dar contexto para el asistente automático

## Proyecto

AriseChatbot es un chatbot de WhatsApp con IA que:
- Recibe mensajes de WhatsApp Cloud API (Meta)
- Procesa los mensajes con IA (Gemini / OpenAI / Ollama)
- Guarda contactos, conversaciones y mensajes en Supabase
- Envía respuestas automáticas de vuelta a WhatsApp

## Flujo principal

1. `app/api/webhook/route.ts` recibe `POST` de Meta
2. `lib/webhook-handler.ts` orquesta el flujo
3. `lib/database-service.ts` guarda y lee historial
4. `lib/ai-service.ts` genera la respuesta con IA
5. `lib/whatsapp-service.ts` envía el mensaje a Meta

## Archivos críticos

- `app/api/webhook/route.ts` — Entrada de webhook y verificación
- `lib/webhook-handler.ts` — Lógica principal de 7 pasos
- `lib/database-service.ts` — Operaciones con Supabase
- `lib/ai-service.ts` — Orquestación de Gemini/OpenAI/Ollama
- `lib/whatsapp-service.ts` — Envío de mensajes WhatsApp
- `AGENT_PROMPT.md` — Prompt del sistema de IA
- `WHATSAPP_ASSISTANT_SKILL.md` — Guía para el asistente Copilot
- `WHATSAPP_UX_STRATEGY.md` — Estrategia de UX y reducción de tokens

## Reglas de cambio

- No cambiar el orden de los 7 pasos sin revisar el flujo completo.
- No hardcodear credenciales en código.
- Validar siempre errores de Supabase y Meta.
- Limitar historial de IA para evitar usar tokens innecesarios.
- Mantener respuestas de IA cortas y directas.

## Variables de entorno importantes

- `WHATSAPP_ACCESS_TOKEN`
- `WHATSAPP_PHONE_NUMBER_ID`
- `WHATSAPP_VERIFY_TOKEN`
- `SUPABASE_URL`
- `SUPABASE_ANON_KEY`
- `GEMINI_API_KEY`
- `OPENAI_API_KEY`
- `OLLAMA_API_URL`
- `AI_BACKEND`

## Cómo usar este asistente

- Si necesitas modificar el comportamiento del bot, usa `AGENT_PROMPT.md`.
- Si necesitas cambiar lógica de procesamiento, ve a `lib/webhook-handler.ts`.
- Si necesitas cambiar env vars, actualiza `.env.local` localmente y también en Vercel si está desplegado.
- Para ver qué herramientas existen, revisa `TOOLS.md`.

## Comandos de Antigravity (Slash Commands)

- `/workflow` — Ejecuta el flujo definido en `docs/ANALYSIS_WORKFLOW.md`.
- `/analyze` — Inicia el análisis técnico profundo usando `ANTIGRAVITY_ANALYZE.skill`.
- `/whatsapp` — Ejecuta la auditoría de WhatsApp y plantillas usando `whatsapp_templates.skill`.

## Resultado esperado

- El asistente debe proporcionar respuestas claras, directas y técnicas.
- El proyecto debe permanecer estable y fácil de mantener.
- Cualquier cambio debe documentarse en los archivos de soporte.
