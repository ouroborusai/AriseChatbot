# Instrucciones del Workspace - AriseChatbot (V2 Industrializada)

Este archivo es la guía principal para cualquier asistente o desarrollador que trabaje en este proyecto.

## Propósito
- Definir el flujo principal de la aplicación.
- Informar de las reglas SSOT (Single Source of Truth).
- Dar contexto de los perfiles y límites de Meta para el asistente automático.

## Reglas Supremas (Ley de Integridad SSOT)
1. **NO SQL Manual:** Prohibido crear scripts SQL sueltos para las plantillas. Hoy, toda plantilla NACE Y MUERE en la carpeta `supabase/templates/*.json`. Si se requiere sincronización en producción, se usa el botón del Dashboard `/api/setup-templates` o se corre `npx tsx scratch/sync_templates.ts`.
2. **Limites de WhatsApp (Meta):** Nunca superes 24 caracteres en los títulos de listas. Meta bota la request sin errores claros.
3. **Manejo de Perfiles:** El orquestador usa `contacts.profile.name` puro del webhook de Meta Cloud API.

## Flujo principal Webhook
1. `app/api/webhook/route.ts`: Extrae payload, mapea perfiles, rechaza duplicados.
2. `lib/webhook-handler.ts`: Pasa la posta a *Contexto Fuerte*. Aplica redundancias dobles, ej: si no encuentra template Id, usa triggers (`hola`).
3. `lib/database-service.ts`: Consolida la magia multi-empresa NxN (`active_company_id`).

## Archivos críticos
- `docs/DATABASE_SCHEMA.md` — Esquema maestro de datos.

## Comandos de Antigravity (Slash Commands)
- Usa los comandos clásicos o simplemente lee el SSOT. 
- *Recuerda*: Cualquier modificación visual a un nodo, se hace editando sus respuestas y dándole a "Auto✨" en el layout. Nada de código duro.
